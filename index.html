Autorización de tratamiento de datos personales

En cumplimiento de la Ley 1581 de 2012 y el Decreto 1377 de 2013 de Colombia, le informamos que los datos personales aquí recopilados (nombre, cédula, teléfono, dirección) serán utilizados únicamente para el registro, control de asistencia y seguimiento pastoral de esta campaña evangelística. Sus datos no serán compartidos con terceros, comercializados ni usados para fines diferentes a los aquí descritos. Usted tiene derecho a conocer, actualizar, rectificar y solicitar la supresión de sus datos en cualquier momento.

Conecta Supabase para que los datos sean reales y permanentes.

Guía de activación con Supabase
1
Ve a supabase.com → crea una cuenta gratis → nuevo proyecto.
2
En el SQL Editor de Supabase, ejecuta el script que aparece abajo para crear las tablas.
3
Ve a Settings → API, copia la Project URL y la anon public key.
4
Pégalos abajo y guarda. El sistema empieza a usar Supabase inmediatamente.
5
Para los correos diarios: crea cuenta en resend.com (gratis), copia la API key y pégala abajo.
6
Sube este archivo HTML a cualquier hosting (GitHub Pages es gratis) para tener una URL pública que usar en el QR.
Ver SQL para crear tablas ▾ 
-- Ejecuta esto en Supabase SQL Editor

CREATE TABLE personas (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  cedula TEXT UNIQUE NOT NULL,
  nombres TEXT NOT NULL,
  apellidos TEXT NOT NULL,
  nombre_completo TEXT GENERATED ALWAYS AS (nombres || ' ' || apellidos) STORED,
  edad INTEGER,
  telefono TEXT,
  iglesia TEXT DEFAULT '',
  direccion TEXT NOT NULL,
  invitador TEXT DEFAULT '',
  creado_en TIMESTAMPTZ DEFAULT now()
);

CREATE TABLE asistencias (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  persona_id UUID REFERENCES personas(id) ON DELETE CASCADE,
  cedula TEXT NOT NULL,
  nombre_completo TEXT,
  fecha DATE NOT NULL DEFAULT CURRENT_DATE,
  hora TIME DEFAULT CURRENT_TIME,
  registrado_en TIMESTAMPTZ DEFAULT now(),
  UNIQUE(persona_id, fecha)
);

-- Índices para búsqueda rápida 5000+ personas
CREATE INDEX idx_personas_cedula ON personas(cedula);
CREATE INDEX idx_asistencias_fecha ON asistencias(fecha);
CREATE INDEX idx_asistencias_persona ON asistencias(persona_id);

-- Habilitar acceso público (Row Level Security)
ALTER TABLE personas ENABLE ROW LEVEL SECURITY;
ALTER TABLE asistencias ENABLE ROW LEVEL SECURITY;

CREATE POLICY "public_insert_personas" ON personas FOR INSERT WITH CHECK (true);
CREATE POLICY "public_select_personas" ON personas FOR SELECT USING (true);
CREATE POLICY "public_insert_asistencias" ON asistencias FOR INSERT WITH CHECK (true);
CREATE POLICY "public_select_asistencias" ON asistencias FOR SELECT USING (true);
CREATE POLICY "public_delete_personas" ON personas FOR DELETE USING (true);
CREATE POLICY "public_delete_asistencias" ON asistencias FOR DELETE USING (true);
